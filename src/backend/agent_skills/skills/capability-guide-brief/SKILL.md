# 能力清单速答

用于回答"这个智能体能做什么、我该怎么问、哪些问题适合问你"。

## Instructions

1. 识别能力咨询意图：如"有哪些能力""能做什么""我该怎么提问""能帮我设计系统吗""支持哪些架构场景"。
2. 先输出能力地图：按 6 类能力分组说明，每类 1-2 句，避免空泛描述。
3. 再给提问模板：按用户关注方向给 8-12 条可直接复制的问题，覆盖模块设计、接口定义、约束验证、方案审查、任务分解等场景。
4. 提供提问技巧：强调系统目标、非功能性约束、规模要求要说清楚。
5. 明确边界：说明哪些设计场景支持完整三步流程，哪些只输出局部设计。
6. 收尾给下一步：给 1 个推荐追问，帮助用户从"了解能力"进入"实际设计任务"。
7. 禁止把能力介绍写成长篇手册；保持结构化、可复制、可执行。

## Inputs

- 用户关于能力范围、使用方式、可提问示例、功能边界的咨询
- 可选：用户关注方向（如模块设计、接口定义、约束分析、方案审查）
- 可选：用户希望的详细度（简版/中版/详细）

## Outputs

- `能力地图（中版）`：
  - **模块架构设计**：根据系统需求识别核心模块，定义每个模块的职责边界，输出符合标准 Schema 的 `modules` 数组。
  - **接口与数据流定义**：基于模块设计定义模块间的调用关系（同步/异步/事件驱动）和完整的数据流转路径。
  - **约束覆盖验证**：逐项验证设计方案是否满足高并发、可扩展性、容错性、模块化四大非功能性约束。
  - **设计方案审查与改进**：对已有设计进行结构完整性检查、一致性检查（悬空引用、孤立模块）和约束覆盖分析，输出问题清单和精炼后的方案。
  - **系统设计任务分解**：将复杂系统设计需求拆解为三步结构化流程（ArchitectAgent→InterfaceAgent→ConstraintAgent），触发 Plan Mode 完整执行。
  - **容量估算与复杂度分析**：给出关键指标的数量级估算（QPS、存储量、带宽）和算法复杂度分析，辅助架构决策。
  - **文档导出**：将设计方案导出为结构化 Word/Excel 报告。
  - **通用知识检索**：通过互联网搜索架构最佳实践、技术选型对比，作为设计决策的参考依据。

- `可直接提问模板（示例）`：
  - 模块设计类：
    - `设计一个高并发短链接系统，帮我拆分核心模块并定义各模块职责。`
    - `一个实时聊天后端需要哪些核心服务模块？请用标准格式输出。`
    - `设计任务调度系统的模块层次，需要支持分布式执行。`
  - 接口定义类：
    - `基于这些模块 [粘贴模块列表]，帮我定义模块间的接口关系和调用类型（同步/异步）。`
    - `短链接系统从用户请求到重定向的完整数据流是什么？`
  - 约束验证类：
    - `验证这个设计 [粘贴 JSON] 是否覆盖了高并发和容错性约束。`
    - `这个系统设计能否支持水平扩展？找出设计中的扩展点和瓶颈。`
  - 方案审查类：
    - `[粘贴设计 JSON] 帮我审查这个架构有什么问题，重点看一致性和约束覆盖。`
    - `这个接口设计中有没有悬空引用或孤立模块？`
  - 任务分解类：
    - `帮我规划设计一个秒杀系统的完整步骤，触发 Plan Mode 逐步执行。`
    - `把"设计分布式消息队列"分解为可执行的设计子任务。`
  - 容量估算类：
    - `日活 100 万的短链接系统，估算 QPS、存储量和带宽需求。`

- `提问技巧`：
  - 明确系统目标：`一个支持 10 万并发的短链接服务` 比 `设计一个系统` 更具体。
  - 说明非功能性约束：`需要 99.99% 可用性`、`读多写少（读写比 10:1）`。
  - 给出规模数量级：QPS、DAU、数据量等，辅助估算和架构决策。
  - 完整任务走 Plan Mode：多步骤设计任务先分解，再逐步执行，每步验证输出。
  - 审查/改进时粘贴完整 JSON：提供完整设计上下文，而非局部片段。

- `边界与可信度说明`：
  - 系统设计输出为结构化 JSON，所有字段均有明确 Schema 约束。
  - 设计结论基于用户提供的需求，不会自行发明未提及的约束或技术栈。
  - 约束覆盖验证只标注设计中有明确对应模块/接口支撑的约束。
  - 容量估算为数量级参考，实际值需结合真实负载测试。

## Response Template

1. `能力地图（6 类）`
2. `可直接复制的提问模板（8-12 条）`
3. `提问技巧（系统目标 + 约束 + 规模）`
4. `边界说明（设计 Schema 约束、估算说明）`
5. `下一步建议（给 1 条可立即执行的追问）`
