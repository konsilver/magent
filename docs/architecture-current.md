## 各层说明

### 客户端（浏览器）
用户直接使用的网页界面，无需安装任何软件。功能包括：左侧会话历史列表、中间对话区域（支持 Markdown 渲染和图表展示）、右侧能力中心（查看可用工具）、底部输入框（支持文字输入和文件上传）。

### 接入层（Nginx）
负责将用户的网页访问请求分发到正确的后端服务，同时托管前端静态页面。用户无感知。

### API 层（FastAPI）
系统的"总调度台"，负责接收前端的所有请求，包括：创建/管理对话、处理流式回复、文件上传解析、用户身份验证、能力目录查询等。同时负责安全校验和日志记录。

### 业务逻辑层（Core）
处理核心业务逻辑，包括：用户认证（支持局内统一账号）、会话和消息的读写、上传文件的存储管理（支持本地、AWS S3 和阿里云 OSS 三种方式）。

### 路由与编排层
根据用户问题的类型，智能决定由哪个智能体来处理：
- **主智能体**：处理日常问答、数据查询、分析等
- **报告生成子智能体**：当检测到"生成报告"类需求时自动切换，专门负责结构化报告的撰写与导出

### AI 推理层（大模型）
经信智能体的"大脑"，根据用户问题类型自动选择合适的大模型：
- **默认模式**：DeepSeek，适合日常问答和数据查询
- **快速模式**：Qwen3-80B，响应更快
- **思考模式**：Qwen3-80B，适合需要深度推理的复杂分析

所有模型都预设了严格的"防幻觉"规则：**必须基于工具返回的真实数据作答，禁止凭空捏造内容**。

### 工具层（6 项 MCP 工具）

| 工具名称 | 数据来源 | 主要用途 |
|---------|---------|---------|
| **数仓查询** | 宁波市工业统计数仓 | 查询规上工业增加值、增速、利润等精确指标 |
| **知识库检索** | Dify 知识库平台 | 检索政策文件、产业报告等文档内容 |
| **互联网搜索** | Tavily 搜索引擎 | 补充查询公开的行业动态、新闻资讯 |
| **产业链信息** | 宁波产业知识中心 | 获取新能源汽车、AI、机器人等产业链全景分析 |
| **图表生成** | 本地渲染（ECharts） | 根据数据自动生成折线图、柱状图、饼图 |
| **报告导出** | 本地生成 | 将分析内容导出为 Word 或 Excel 文件 |

### 数据存储

| 存储类型 | 技术 | 存储内容 |
|---------|------|---------|
| 关系型数据库 | PostgreSQL | 用户信息、对话历史、消息记录、反馈数据 |
| 文件/对象存储 | 本地 / S3 / 阿里云 OSS | 用户上传的文件、智能体生成的报告和图表 |

### 运行监控
后台自动运行，工作人员无需操作。包括：
- **日志收集**：记录所有操作便于排查问题
- **性能指标**（Prometheus + Grafana）：实时监控系统运行状态
- **链路追踪**（Jaeger）：追踪每次对话的完整处理过程

## 一次对话的完整流程

```
1. 你在网页输入问题，点击发送
         ↓
2. 系统自动判断问题类型
   └─ 普通问答 → 主智能体处理
   └─ 生成报告 → 报告子智能体处理
         ↓
3. 大模型调用相关工具查询真实数据
   ├─ 数仓  →  获取工业统计数字
   ├─ 知识库 → 检索政策/报告文档
   ├─ 互联网 → 搜索公开资讯
   └─ 产业链 → 获取产业全景数据
         ↓
4. 大模型基于工具返回的真实数据生成回答
   （无数据 → 如实告知，不编造内容）
         ↓
5. 回答流式实时推送到你的屏幕
   ├─ 文字内容（Markdown 格式）
   ├─ 数据图表（自动展示）
   └─ 可下载的 Word / Excel 文件（附件区）
```

## 部署方式

系统通过 **Docker** 容器化部署，共 7 个服务：

| 服务 | 说明 | 访问地址 |
|------|------|---------|
| 前端页面 | React 网页应用 | `:3002` |
| 后端服务 | FastAPI 接口服务 | `:3001` |
| 数据库 | PostgreSQL | `:5432`（内部） |
| 监控看板 | Grafana | `:3003` |
| 指标采集 | Prometheus | `:9090`（内部） |
| 链路追踪 | Jaeger | `:16686`（内部） |
| 告警服务 | AlertManager | `:9093`（内部） |
