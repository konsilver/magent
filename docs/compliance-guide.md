# 合规性指南

## 概述

本文档说明 Jingxin-Agent 在数据保护、隐私、安全等方面的合规性策略和实践。

**适用法规**:
- GDPR (General Data Protection Regulation) - 欧盟通用数据保护条例
- PIPL (Personal Information Protection Law) - 中国个人信息保护法
- CCPA (California Consumer Privacy Act) - 加州消费者隐私法

---

## 1. GDPR 合规性

### 1.1 个人数据定义

根据 GDPR，个人数据是指任何与已识别或可识别的自然人相关的信息。

**Jingxin-Agent 收集的个人数据**:
- ✓ 用户 ID（来自用户中心）
- ✓ 用户名
- ✓ 邮箱地址
- ✓ 头像 URL
- ✓ IP 地址（日志）
- ✓ 设备信息（User-Agent）
- ✓ 会话内容（聊天记录）
- ✓ 上传的文件和文档

**不收集的敏感数据**:
- ✗ 密码（由用户中心管理）
- ✗ 身份证号
- ✗ 银行卡号
- ✗ 健康数据
- ✗ 种族、宗教、政治观点

### 1.2 合法性基础 (Lawful Basis)

根据 GDPR 第 6 条，数据处理必须有合法基础。

**我们的合法基础**:
- **合同履行**: 提供 AI 助手服务需要处理用户数据
- **合法利益**: 日志记录用于安全和服务改进
- **用户同意**: 在用户注册时获得（由用户中心管理）

### 1.3 数据最小化 (Data Minimization)

**原则**: 只收集和处理必要的数据

**实施**:
- ✓ 仅存储服务必需的用户信息（user_id, username, email）
- ✓ 不收集无关的个人信息
- ✓ 审计日志只记录操作类型，不记录完整请求内容
- ✓ 敏感字段（密码、Token）在日志中自动脱敏

### 1.4 数据准确性 (Accuracy)

**原则**: 确保个人数据准确和最新

**实施**:
- ✓ 用户信息与用户中心同步（每次登录时更新）
- ✓ 用户可通过用户中心更新个人信息
- ✓ `users_shadow.last_sync_at` 记录同步时间

### 1.5 存储期限限制 (Storage Limitation)

**原则**: 数据不应保留超过必要期限

**实施**:
- ✓ 软删除的会话保留 30 天后永久删除
- ✓ 审计日志保留 90 天
- ✓ 临时文件在处理完成后立即删除
- ✓ 用户注销后，个人数据在 90 天后匿名化或删除

**数据保留策略**:

| 数据类型 | 保留期限 | 删除方式 | 依据 |
|---------|---------|---------|------|
| 用户资料 | 账号存续期间 + 90天 | 匿名化或删除 | 合同履行 |
| 聊天会话 | 用户主动保留 + 30天（软删除后） | 永久删除 | 用户控制 |
| 聊天消息 | 随会话删除 | 级联删除 | 关联数据 |
| 上传文件 | 用户主动保留 + 30天（删除后） | 物理删除 | 用户控制 |
| 审计日志 | 90 天 | 归档或删除 | 合规要求 |
| 访问日志 | 30 天 | 滚动删除 | 安全需要 |

### 1.6 数据主体权利 (Data Subject Rights)

根据 GDPR 第 15-22 条，用户有以下权利：

#### a) 访问权 (Right to Access)

**用户可以请求访问其个人数据**

实施方式:
- API 端点: `GET /v1/me` - 查看个人信息
- API 端点: `GET /v1/chats` - 查看所有会话
- API 端点: `GET /v1/audit/logs` - 查看操作日志

#### b) 更正权 (Right to Rectification)

**用户可以请求更正不准确的数据**

实施方式:
- 用户通过用户中心更新个人信息
- Jingxin-Agent 在下次登录时同步更新

#### c) 删除权 (Right to Erasure / "Right to be Forgotten")

**用户可以请求删除其个人数据**

实施方式:

**1. 会话删除**:
```bash
DELETE /v1/chats/{chat_id}
```
- 软删除会话，30天后永久删除
- 级联删除所有关联消息

**2. 文件删除**:
```bash
DELETE /v1/files/{file_id}
```
- 从对象存储删除文件
- 删除数据库记录

**3. 账号删除**:
- 用户通过用户中心注销账号
- Jingxin-Agent 在 90 天后：
  - 匿名化用户资料（保留统计）
  - 删除所有会话和消息
  - 删除所有文件
  - 保留审计日志（匿名化 user_id）

实施计划: ⚠️ **待实现** - 账号完全删除功能 (P2 优先级)

#### d) 限制处理权 (Right to Restriction of Processing)

**用户可以请求限制数据处理**

实施方式:
- 用户可以暂停服务使用（由用户中心管理）
- 数据仍保留，但不进行新的处理

#### e) 数据可携带权 (Right to Data Portability)

**用户可以请求以结构化、常用格式导出数据**

实施方式:

**导出个人数据**:
```bash
GET /v1/export/my-data
```

导出内容（JSON 格式）:
- 用户资料
- 所有会话和消息
- 上传的文件列表
- 操作日志

实施计划: ⚠️ **待实现** - 数据导出功能 (P2 优先级)

#### f) 反对权 (Right to Object)

**用户可以反对某些数据处理**

实施方式:
- 用户可以选择退出数据分析（如果实施）
- 用户可以撤回同意（通过用户中心）

### 1.7 数据安全 (Security of Processing)

**GDPR 第 32 条要求采取适当的技术和组织措施**

实施:
- ✓ 传输加密: HTTPS/TLS
- ✓ 存储加密: 数据库连接加密，S3 server-side encryption
- ✓ 访问控制: 基于 Token 的认证和授权
- ✓ 日志脱敏: 敏感信息自动掩码
- ✓ 审计日志: 所有关键操作可追踪
- ✓ 定期安全评估: 季度安全审查
- ✓ 事件响应: 数据泄露响应流程（见下文）

### 1.8 数据泄露通知 (Data Breach Notification)

**GDPR 第 33-34 条要求在 72 小时内通知监管机构**

**数据泄露响应流程**:

1. **检测和确认** (0-4 小时)
   - 确认泄露范围
   - 评估影响的数据类型和用户数量
   - 隔离受影响系统

2. **遏制和恢复** (4-24 小时)
   - 停止泄露
   - 修复漏洞
   - 恢复正常服务

3. **通知** (24-72 小时)
   - 通知 DPO (Data Protection Officer)
   - 通知监管机构（如果适用）
   - 通知受影响用户（如果高风险）

4. **事后分析** (1 周内)
   - 根因分析
   - 改进措施
   - 更新应急预案

**通知模板**: 见 `docs/data-breach-notification-template.md`

### 1.9 数据处理记录 (Records of Processing)

**GDPR 第 30 条要求维护处理活动记录**

| 处理活动 | 数据类型 | 目的 | 合法基础 | 保留期限 | 接收方 |
|---------|---------|------|---------|---------|-------|
| 用户认证 | user_id, username, email | 提供服务 | 合同履行 | 账号存续期 | 内部系统 |
| 聊天服务 | 会话内容, 消息 | AI 助手对话 | 合同履行 | 用户控制 | 模型 API (匿名化) |
| 文件存储 | 上传文件, 元数据 | 知识库功能 | 合同履行 | 用户控制 | 对象存储 |
| 审计日志 | 操作记录, IP地址 | 安全和合规 | 合法利益 | 90 天 | 内部系统 |

## 2. 中国个人信息保护法 (PIPL) 合规性

### 2.1 个人信息处理规则

**告知同意原则**:
- ✓ 用户注册时明确告知收集的信息类型和用途
- ✓ 隐私政策清晰可访问
- ✓ 获得用户明确同意

**最小必要原则**:
- ✓ 仅收集必要的个人信息
- ✓ 不收集无关数据

### 2.2 敏感个人信息

**PIPL 定义的敏感个人信息**:
- 生物识别
- 宗教信仰
- 特定身份
- 医疗健康
- 金融账户
- 行踪轨迹

**Jingxin-Agent 处理**:
- ✗ 不收集任何敏感个人信息
- ✓ 如果未来收集，需要单独同意

### 2.3 个人信息跨境传输

**如果数据传输到境外，需要满足以下条件之一**:
1. 通过国家网信部门的安全评估
2. 经专业机构的个人信息保护认证
3. 与境外接收方订立合同，约定双方权利义务
4. 其他法律规定的条件

**当前状态**:
- ✓ 数据存储在中国境内（默认配置）
- ⚠️ 如果使用境外模型 API，需要评估合规性

**境外传输清单**:

| 服务 | 数据类型 | 目的国 | 保护措施 |
|------|---------|-------|---------|
| OpenAI API | 对话内容（匿名化） | 美国 | 数据处理协议, 匿名化 |
| 其他模型 API | 对话内容（匿名化） | - | 根据服务商 |

**注**: 传输给模型 API 的数据应匿名化，不包含可识别个人的信息。

### 2.4 个人信息保护影响评估

**以下情况需要进行影响评估**:
- 处理敏感个人信息
- 利用个人信息进行自动化决策
- 委托处理个人信息、向第三方提供或公开个人信息
- 向境外提供个人信息

**Jingxin-Agent 评估结果**:
- ✓ 不处理敏感信息，无需评估
- ⚠️ 如果提供给境外模型 API，需要评估

## 3. CCPA 合规性 (加州消费者隐私法)

### 3.1 消费者权利

**CCPA 赋予加州消费者的权利**:
- ✓ 知情权: 了解收集了哪些个人信息
- ✓ 删除权: 要求删除个人信息
- ✓ 选择退出权: 拒绝出售个人信息
- ✓ 不歧视权: 行使权利不受歧视

**Jingxin-Agent 遵守**:
- ✓ 不出售用户个人信息
- ✓ 提供数据访问和删除功能
- ✓ 隐私政策明确说明收集的数据

## 4. 隐私政策要点

### 4.1 必须包含的内容

**隐私政策应包含**:
1. 收集哪些个人信息
2. 收集信息的目的
3. 如何使用信息
4. 与谁共享信息
5. 保留多长时间
6. 用户的权利
7. 如何联系我们

### 4.2 隐私政策模板

```markdown
# Jingxin-Agent 隐私政策

## 1. 我们收集的信息

- 账号信息: 用户名、邮箱（来自用户中心）
- 使用信息: 会话记录、上传文件
- 日志信息: IP 地址、设备信息

## 2. 我们如何使用信息

- 提供 AI 助手服务
- 改进服务质量
- 安全和反欺诈
- 遵守法律义务

## 3. 信息共享

我们不会出售您的个人信息。
我们可能与以下方共享信息:
- 云服务提供商（AWS S3, 数据库托管）
- AI 模型提供商（对话内容，匿名化）

## 4. 数据保留

- 账号信息: 账号存续期 + 90 天
- 聊天记录: 您主动删除后 30 天
- 审计日志: 90 天

## 5. 您的权利

- 访问您的数据
- 更正不准确的数据
- 删除您的数据
- 导出您的数据

## 6. 安全措施

- HTTPS 加密传输
- 数据库访问控制
- 日志脱敏
- 定期安全审计

## 7. 联系我们

如有隐私问题，请联系: privacy@example.com
```
