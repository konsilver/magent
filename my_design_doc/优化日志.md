# 优化1（Warmup 并行预取 Step 0）
将原来的串行 await _run_warmup(...) + 后续再查 step 0 记忆，改为 asyncio.gather 同时跑，Step 0 的 Milvus 查询在 Warmup LLM 调用期间完成，不额外占时间。

# 优化2（每步预取下一步记忆）
在消费当前步记忆后、启动 SubAgent 前，立刻用 asyncio.ensure_future 异步发出下一步的记忆查询。SubAgent 执行（LLM 调用，通常数秒）期间 Milvus 查询已完成，下一步进来直接 await 即可拿到结果。
